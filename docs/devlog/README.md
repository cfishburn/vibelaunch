# Devlogs

This directory is the working home for active project devlogs.

Use the public standard in [`src/content/superpowers/dev-log.mdx`](../../src/content/superpowers/dev-log.mdx) as the template, but keep real task logs here instead of the public content collection.

Rule of thumb:

- create a devlog when a task spans multiple sessions or has meaningful tradeoffs
- skip it for trivial edits and obvious fixes
- promote lasting lessons into durable docs when the work settles
